Xiaomi Mi Router AX3000T v2 (RD03v2) OpenWrt port — overlay manifest
=====================================================================
Base:   openwrt/openwrt @ 25ee12629edcc38feffbd06255dd47840cd7af7e
        ("uboot-tools: update to v2026.07")
Device: IPQ5018 SoC (2.4 GHz radio) + QCN6122 (5 GHz) + Airoha AN8855 4-port switch
Usage:  ./build.sh  (clones OpenWrt, checks out 25ee126, overlays files/, builds)

Every path below is relative to files/ and lands at the same path inside
the OpenWrt tree. Sizes in bytes. [bin] = binary, keep byte-identical.

Deliberately excluded: target/linux/qualcommax/ipq60xx/config-default and
ipq807x/config-default diffs from the source tree — they only add
"# CONFIG_*AN8855* is not set" housekeeping for those sibling subtargets
(new Kconfig symbols introduced by our kernel patches default to n anyway).
Not needed for an ipq50xx-only build.

    960  package/boot/uboot-tools/uboot-envtools/files/qualcommax_ipq50xx
         Adds xiaomi,mi-router-ax3000t-v2 to the uboot-envtools device list
         (env in 0:APPSBLENV @ 0x0, size 0x10000, sector 0x20000). Without it
         /etc/fw_env.config is never generated, fw_setenv fails, and sysupgrade
         cannot set U-Boot's dual-boot flags -- the NAND boot then loops
         ("Boot failure on both systems"). See docs/nss-offload.md.

NSS hardware-offload overlay (nss/) — applied only with `NSS=1 ./build.sh`
=====================================================================
Experimental; see docs/nss-offload.md. Layered on top of files/:

        nss/feeds.conf.append   Extra feeds: qosmio nss-packages + sqm-scripts-nss.
        nss/config.append       NSS kernel symbols (skb_recycler, conntrack DSCP-remark,
                                and NET_DSA_TAG_VSC73XX_8021Q for the offload DSA tagger).
        nss/feed-patches/qca-nss-drv/0029-ipq50xx-nss-core-boot-reset-pulse.patch
                                The IPQ5018 NSS core-boot fix (reset-pulse), applied to
                                the qca-nss-drv feed package after `feeds update`.
        nss/feed-patches/qca-nss-ecm/0026-frontends-nss-dsa-conduit-awareness.patch
                                ECM DSA front-end: maps a DSA user port (or a bridge
                                master over one) to its CPU conduit so the fast path
                                resolves an accelerable NSS interface.
        nss/overlay/            Path-preserving overlay (same layout as files/):
          patches-6.12/06xx,0819,0981  ECM/clients/skb_recycler + NSS reserved-memory.
          files/.../ipq5018-nss.dtsi   NSS core node (CSM regs, 8 IRQs, 8 MB DDR @0x40000000).
          files/net,include/           skb_recycler + conntrack DSCP-remark sources.
          files/.../drivers/net/dsa/an8855.c  tag_8021q build of the AN8855 driver (overrides
                                       the files/ MTK-tag build so the NSS parser sees the
                                       EtherType, not the special tag).
          ipq50xx/.../board.d/02_network  Gateway topology: moves the WAN onto the eth0 CPU
                                       port (offload needs WAN and LAN on different CPU ports).
          ipq50xx/.../etc/rc.local     Enables NSS redirect + ipv{4,6}_accel_mode at boot, and
                                       re-arms U-Boot's dual-boot flags (flag_boot_success +
                                       flag_try_sys*_failed) so the boot failsafe never trips.
          ipq50xx/.../etc/uci-defaults/00-nss-manual  Disables the qca-nss init scripts so
                                       they don't double-load the autoloaded modules.
          etc/modules.d/33-qca-nss-ecm ECM autoload at boot (pulls in qca-nss-drv, boots core).
        build.sh also appends the NSS packages to the AX3000T DEVICE_PACKAGES
        and #includes ipq5018-nss.dtsi into the device DTS.
