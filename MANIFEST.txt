Xiaomi Mi Router AX3000T v2 (RD03v2) OpenWrt port — overlay manifest
=====================================================================
Base:   openwrt/openwrt @ 25ee12629edcc38feffbd06255dd47840cd7af7e
        ("uboot-tools: update to v2026.07")
Device: IPQ5018 SoC (2.4 GHz radio) + QCN6122 (5 GHz) + Airoha AN8855 4-port switch
Usage:  ./build.sh  (clones OpenWrt, checks out 25ee126, overlays files/, builds)

Every path below is relative to files/ and lands at the same path inside
the OpenWrt tree. Sizes in bytes. [bin] = binary, keep byte-identical.

Deliberately excluded: target/linux/qualcommax/ipq60xx/config-default and
ipq807x/config-default diffs from the source tree — they only add
"# CONFIG_*AN8855* is not set" housekeeping for those sibling subtargets
(new Kconfig symbols introduced by our kernel patches default to n anyway).
Not needed for an ipq50xx-only build.

WiFi low-memory support (ath11k smallbuffers — 256 MB fits two radios)
----------------------------------------------------------------------
   1791  package/kernel/mac80211/patches/ath11k/951-wifi-ath11k-introduct-CONFIG_ATH11K_SMALLBUFFERS.patch
         Ziyang Huang's patch (OpenWrt PR #21495, authorship preserved):
         gates ath11k's DP ring-buffer sizes behind CONFIG_ATH11K_SMALLBUFFERS
         so they shrink on low-RAM devices. Halves the ath11k footprint.
  12707  package/kernel/mac80211/Makefile
         Adds -DCONFIG_ATH11K_SMALLBUFFERS to the smallbuffers build variant
         (mirrors the existing ath10k-smallbuffers define).
  15461  package/kernel/mac80211/ath.mk
         Splits kmod-ath11k into regular + smallbuffers variants
         (kmod-ath11k-smallbuffers, PROVIDES kmod-ath11k). Our device
         selects the smallbuffers variant in ipq50xx.mk DEVICE_PACKAGES.
